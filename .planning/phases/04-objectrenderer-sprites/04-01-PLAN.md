This task has two parts: (A) fix the SPRITES constant region values, and (B) add renderWidth/renderHeight and fix drawOffsetY on every furniture.push() call.

**CRITICAL NOTE on drawOffsetY sign convention:** The engine applies `worldY = tileY * tileSize - (drawOffsetY ?? 0)`. Positive drawOffsetY shifts UP. The current file has NEGATIVE values (e.g., `-48`, `-16`, `-8`) which shift sprites DOWN -- this is WRONG for all cases. All values must be changed to their positive equivalents. The formula is: `drawOffsetY = renderHeight - heightTiles * tileSize`. When renderHeight equals the footprint pixel height, drawOffsetY = 0.
